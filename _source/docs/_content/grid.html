--- title: Grid order: 01 ---

.grid

.grid is a flexible, responsive, tiny utility class for composing pages. Twelve columns plus two “shadow” columns for edge-to-edge content keep the system simple, but potent. At just 2KB of CSS, .grid can produce most common page layouts, and a bunch of uncommon ones too. You’ll write fewer media queries, think less about mobile-specific styles, and spend more time focused on the key, unique parts of the page.

.grid is fluid and responsive. Columns and gutters scale alongside your type as window dimensions change in order to maintain layout consistency and reduce the number of breakpoints required (grid uses three). At the smallest screen sizes .grid collapses to a one column layout, but keeps the two shadow columns in order to preserve full-bleed page elements. Simple, predictable, and effective.

.grid is nestable, which solves two issues:

  1. Maintaining a consistent layout grid without having to “flatten” your markup.
  2. Creating full-bleed page breakouts with content that aligns to the same grid columns.

Nest grids to create full-bleed breakouts with grid-aligned content, like this one.

.grid is not designed for generated repeating content. You can make it happen with conditionals in your template code, but you’re better off creating a separate block class, then using this system to control size and placement on the page.

.grid is not designed for older browsers. It uses CSS grids under the hood, with no fallbacks or polyfills. A fallback solution is highly dependent on how much fidelity you’re willing to lose, so it’s up to you to implement a solution that aligns with your project goals.

.grid__item

.grid__item is the basic building block of the grid system. Without modifiers, grid items span six columns starting on column 2. All direct children of .grid are mapped to these same columns too; both a sensible default, and a means to style markup that you can’t easily add classes too (Markdown generated markup, for instance).

--span modifiers

--span-[3-12] modifiers alter the number of columns a grid item spans. --span-12 takes the additional step of setting the start column to one since it needs the full width.

--start modifiers

--start-[1-9] modifiers change the start column to the corresponding number.

--start-auto makes a grid item start in the next unoccupied column, assuming there’s enough room for it. Practically speaking, it means you don‘t have to do math when laying out multiple grid items in a single row. Give the first grid item a specific start column, then use --start-auto for the rest.

--bleed modifiers

--bleed-[left/right/full] modifiers break elements out of the grid and pin their sides to the left and right page edges. Edges stay pinned no matter how large or small the viewport. Full-bleed, guaranteed.

grid__item --span-7 --bleed-left

grid__item --start-7 --bleed-right

grid__item --bleed-full

Examples

When you put it all together, you can create an immense variety of page layouts. Always beautifully responsive. Just 2KB. Combine it with other utility classes like .space and .text and you’d be surprised at how much you can do with so little CSS.
