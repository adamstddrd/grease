.animate

.animate is a simple system for rapidly creating complex transitions. Create a CSS keyframe animation, and use .animate to apply it to any element and manage when it runs. Progressive enhancement is baked in, so you don’t have to worry about leaving your visitors with a broken page if they have js disabled.

--[animation-name] modifiers

.animate ships with a single animation modifier, --fade-in. It’s both a useful animation and a proof of concept for you to follow. The idea is to give you the tools to create your own animations that are flexible and composable vs. packing in dozens of animations that you probably won’t use.

To add a new animation, create your keyframe animation and add a corresponding modifier with the same name in _source/_assets/stylesheets/utilities/animate.css.

--delay modifiers

--delay modifier change your animations to run after a delay instead of immediately.

animate --fade-in --delay-s

animate --fade-in --delay-m

animate --fade-in --delay-l

animate --fade-in --delay-xl

animate --fade-in --delay-xxl

--duration modifiers

Any animation modifier you create should include a default duration. The --duration modifiers let you alter the duration on a case-by-case basis.

animate --fade-in --duration-s

animate --fade-in --duration-m

animate --fade-in --duration-l

animate --fade-in --duration-xl

animate --fade-in --duration-xxl

Scroll-triggered animations

By default, animations play when the element is added to the page, whether it’s in view or not. That’s great for header animations, but what about content further down the page? Simply move the animation modifier from the class property to a data attribute named data-aos and the animation won’t run until the element enters the viewport. You can combine this with the delay modifiers too.

class="animate --delay-xl" data-aos="--fade-in"

It's not always practical or possible to add animation properties to every element you'd like to animate, so .animate comes with a method for animating child elements. Add your animation properties to the parent element instead along with a data-aos-children attribute, and the animation will be removed from the parent and copied to each child element at runtime.
